# Contributing

## Workflow
1. Pull the latest `main`
2. Create your branch
3. Make small changes
4. Commit often
5. Push your branch
6. Open a pull request
7. Respond to review feedback in the same PR

## Local checks before pushing
- App starts with `npm run dev`
- Search works
- Filter works
- No obvious console errors
- Your branch only contains the work for this assignment

## Notes
- Keep all week 1 data mocked
- Do not rewrite the starter structure unless your PR clearly explains why
- If you change the data shape, update the related UI code too
